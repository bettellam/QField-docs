    QField同步插件¶

    QField同步插件帮助为QField准备和打包QGIS工程。

    QField同步通过以下自动化操作支持工程准备工作：

    • 工程设置所需的步骤 (例如 portable_project)
    • 从单个栅格图层或从地图主题中定义的样式 创建底图。
    • 配置离线编辑功能和同步 回更改。

    工作流程¶

    快速浏览一下工作流程，主要包含以下典型步骤：

    1. 创建QField包。这是一个工作副本在单独 文件夹。
    2. 将QField包复制到目标设备。
    3. 到外业采集数据。
    4. 将修改的数据拷回计算机。
    5. 将修改的数据与数据库或文件同步。

    安装¶

    在QGIS中打开插件库并搜索 qfield sync ，在列表中选择插件并单击 安装 。

    配置¶

    工程配置信息保存在 .qgs 主文件中，这样可以预先配置工程文件并重复使用。

    图层配置¶

    在工程配置对话框中，可以分别为每个图层定义一个 动作 。根据图层类型，可以使用不同类型的动作。

    复制

    该图层将复制到包文件夹。这仅适用于基于文件的图层。

    什么也不做

    图层源将保持不变。这仅适用于非基于文件的图层，如 WMS、WFS、Postgis 等。

    离线编辑

    图层的副本将复制到包文件夹。工作期间在打包工程中所做的每个更改都记录在更新日志中。后期进行数据同步时，日志中记录的操作将被重新执行，所有的更新都将被同步到主数据中。

    移除

    该图层将从工作副本中移除。如果在底图中使用了某个图层，并且在打包工程中不可用，则此选项非常有用。

    配置底图¶

    底图是栅格图层，将作为最底图层添加到打包的工程文件中。

    如果启用了底图选项，无论何时打包工程，都将渲染底图。感兴趣的区域 (将渲染的范围) 在打包时进行选择。

    底图有两种可能的来源：

    图层

    栅格图层。这对于获取在线图层 (如WMS) 的离线副本，或处理不支持格式 (如 ECW 或 MrSID 图层) 的工作副本很有帮助。

    笔记

    目前iOS设备不支持JPEG栅格数据集 (包括JPEG编码的MBTiles)。

    地图主题

    地图主题。这对于基于多个样式图层的组合而创建的底图非常有用。之后，这些图层可以从工作包中移除，不需要在设备上渲染。从而节省设备上的磁盘空间和电池消耗。

    瓦片大小定义了空间分辨率，它确定了每个像素的贴图单位数。如果地图画布CRS以米为单位，并且瓦片大小设置为 1，则每个栅格像素的空间范围是 1x1 米；如果设置为 1000，则每个栅格像素的空间范围将为 1 平方千米。

    离线编辑配置¶

    如果勾选了 仅同步感兴趣区域中的要素 ，则仅将打包时在地图画布范围内的要素复制到离线编辑的工作副本中。

    为QField打包¶

    配置工程之后，将其打包到文件夹中。此文件夹将包含 .qgs 和其中使用的数据。

    将文件夹复制到设备中。启动QField，打开工程就可以采集数据了。

    还要确保使用QGIS的 "另存为" 来保存工程，因为稍后需要同步更改时必须重新打开该工程。

    如何在没有QField云的情况下从iOS设备同步或同步到iOS设备¶

    使用iTunes的文件共享功能导入到QField根文件夹。

    1. 打开iTunes应用程序，单击iTunes窗口左上方的iPhone按钮。
    2. 单击左侧边栏中的 文件共享 选项。
    3. 选择应用程序QField并单击 添加文件 ，将打开文件浏览器。
    4. 选择文件。

    从QField同步¶

    当您需要同步收集内容时，请在QGIS中重新打开工程 (您用 "另存为" 保存的工程)。

    将工程文件夹从设备复制到计算机，然后使用 从QField同步 菜单将可移植工程的更改同步到主工程中。

    请确保只同步一次数据。这意味着，如果您再次外出收集更多数据时，应该在此之前创建新的QField包，避免以后同步出现问题 (例如重复数据等)。
